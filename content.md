# content.md

This file is a content scaffold for `koorevaar.com`.

It is not the final website copy. It exists to help generate the first version of the site structure and text. Content can be filled in gradually.

## General content direction

The site should present Patrick Koorevaar as a DevOps Engineer with a strong background in Azure, AKS, CI/CD, Kubernetes, and infrastructure automation.

The tone should be:

- professional
- modern
- practical
- technical but still readable
- confident without exaggeration

The content should emphasize:

- current DevOps role
- practical engineering experience
- side projects used for learning and experimentation
- modern development workflow
- low-cost but thoughtful architecture choices

## Content rules

- Prefer concrete language over vague buzzwords.
- Keep paragraphs short.
- Do not overstate experience.
- Focus on practical engineering and learning.
- Avoid filler marketing phrases.
- The homepage should feel professional first, experimental second.

## Generation notes

When generating content from this file:

- use these sections as structure
- keep the first version concise
- use placeholders where real content is still missing
- do not invent achievements or roles
- prefer clean summary text over long detailed prose
